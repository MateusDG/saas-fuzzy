Visão geral

O que este MVP resolve agora

O Kouzina Reco é um MVP de recomendação para e-commerce premium. A versão atual prioriza o fluxo mensurável: widget instalado, API respondendo, eventos anônimos registrados, catálogo importado e ranking v0 explicável.

01

Produto funcional

O widget identifica contexto de produto, busca recomendações e renderiza cards no demo local.

02

Medição de uso

A API aceita `page_view`, `product_view`, `recommendation_impression` e `recommendation_click`.

03

Catálogo e ranking

Produtos seedados ou oficiais autorizados alimentam um ranking v0 por regras comerciais.

Futuro

Fuzzy e ontologia

Fluxo do widget

Da página ao clique medido

Este é o caminho operacional do `widget/kouzina-reco.js` no demo local.

  1. 01detecta `product_id`

    Lê container, meta tag ou texto com código do produto.

  2. 02cria `anonymous_id`

    Usa `localStorage` com fallback em memória se o navegador bloquear.

  3. 03cria `session_id`

    Usa `sessionStorage` para separar sessões sem identificar pessoa.

  4. 04envia `page_view`

    Registra carregamento do widget e contexto da página.

  5. 05busca recomendações

    Chama `GET /recommendations` com produto e widget.

  6. 06renderiza cards

    Mostra nome, preço quando existe, imagem quando existe e razão textual.

  7. 07envia impressão

    Registra `recommendation_impression` com quantidade e IDs recomendados.

  8. 08envia clique

    Registra `recommendation_click` com o produto recomendado clicado.

Endpoints

Contrato público atual da API

Três endpoints sustentam o MVP: saúde do serviço, recomendações e eventos anônimos.

real com banco

GET /health

Confirma que a API FastAPI está respondendo.

Entrada
Sem payload.
Resposta
{"status":"ok","service":"kouzina-reco-api","version":"0.1.0"}
dependente do catálogo

GET /recommendations

Busca produto no catálogo e aplica ranking v0; se não houver base suficiente, usa fallback.

Query
product_id=119&widget_id=product-page
Resposta
{"widget_title":"Complete seu projeto","recommendations":[...]}
real com banco

POST /events

Valida eventos do widget e persiste em `recommendation_events` quando o banco está disponível.

Payload
{"event_type":"recommendation_click","anonymous_id":"frontend_visual_anon"}
Resposta
{"received":true,"event_type":"recommendation_click","timestamp":"..."}

Modelo de dados

Tabelas que sustentam o MVP

O modelo permanece simples: loja, produtos, relações manuais futuras e eventos do widget.

stores

Loja padrão `kouzina` e chave pública do widget.

  • public_key
  • created_at

Produtos seedados ou importados do catálogo oficial autorizado.

  • image_url

  • price
  • product_type
  • environment

manual_product_relations

Eventos anônimos para medição do widget, sem dados pessoais.

Privacidade e LGPD

Minimização de dados desde o MVP

O projeto mede comportamento do widget sem identificar a pessoa visitante.

Permitido

  • `anonymous_id`
  • `session_id`
  • `event_type`
  • `page_url`
  • `product_id`
  • `widget_id`
  • `recommended_product_id`
  • `metadata` técnico não sensível

Proibido

  • nome
  • CPF
  • e-mail
  • telefone
  • endereço
  • pagamento
  • conteúdo de formulário
  • mensagens de WhatsApp

Catálogo e importação

Como os produtos chegam ao ranking

A importação é local, autorizada e idempotente por `external_id`.

local

data/products_seed.csv

Catálogo fictício ou semi-realista para desenvolvimento e validação sem depender do site real.

ignorado pelo Git

data/products_kouzina_official.csv

Snapshot oficial autorizado, local, com importação por CSV UTF-8 com BOM e separador `;`.

regra ativa

Seed idempotente

O importador cria ou atualiza produtos por `external_id`, evitando duplicidade.

catálogo

Preço e imagens

`0.00` vira “Sob consulta”; imagens são URLs salvas no banco, não binários.

demo

Produto 119

O demo local usa `product_id=119` quando o catálogo oficial autorizado está importado.

Revisão qualitativa

Validação comercial antes de métricas avançadas

O arquivo `reports/recommendation_review.csv` reúne produtos de origem, recomendações, score, razão textual e campos para avaliação manual da Kouzina.

geração local

reports/recommendation_review.csv

Serve para revisar se as recomendações fazem sentido comercial antes de CTR, fuzzy, ontologia ou ajustes automáticos.

current_product recommended_product score reason reviewer_rating reviewer_comment
